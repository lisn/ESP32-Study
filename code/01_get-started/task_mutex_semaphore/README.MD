二进制信号量与互斥信号量的区别

二进制信号量，主要用于同步，同步的二进制信号量在成功获取（获得）后，**不需要**归还，任务同步是通过让一个任务或中断“给出”信号量，而另一个任务“获取”信号量来实现的。

互斥量（Mutexes）， 当一个持有互斥量的任务被另一个更高优先级的任务试图获取同一个互斥量时，其优先级将被提升。此时，可以说原本持有互斥量的任务"继承"了那个试图"获取"同一互斥量的任务的优先级。当互斥量被归还时，这种继承的优先级将被"解除"（即该任务在持有互斥量期间所继承的较高优先级会恢复为其原始优先级）。

用于实现互斥访问的任务在获取互斥量后，**必须始终**将其归还——否则其他任何任务都将永远无法获取该互斥量。



递归互斥量：任务A，多次获得互斥量，必须释放相同的资料，任务B才能获得，否则任务B不能获得。
